## Database Design & Supabase Setup
Supabase project initialization with PostgreSQL schema for all 8 core tables (users, portfolio, categories, followed_topics, articles, user_news_feed, alerts, reports). GIN indexes on array columns, row-level security policies, and seed data for the 3-level category hierarchy.

## Authentication Implementation
User registration and login with bcrypt password hashing, JWT token generation via FastAPI, 24-hour token expiry, and Supabase auth integration. Flutter login/register screens with form validation.

## Portfolio Management (Backend + Frontend)
FastAPI endpoints for adding, updating, and deleting portfolio assets with Yahoo Finance API validation. Auto-subscription to relevant categories on asset addition. Flutter portfolio screen with asset cards, quantity/price input, and real-time price display.

## News Collection Pipeline & AI Integration
Celery scheduled task for fetching news from MediaStack API every 15 minutes. Duplicate filtering by URL. Active asset pool generation from all user portfolios. OpenAI GPT-4o-mini API integration with the constrained prompt template for sentiment analysis, categorization, and multi-asset impact scoring.

## AI Prompt Implementation & Response Processing
Fine-tuning the prompt template for consistent JSON responses. Implementing response validation (JSON parsing, asset pool check, category check, severity clamping). Retry logic for failed or malformed AI responses. Storing processed articles with AI-generated fields in Supabase.

## Feed Distribution & News Feed UI
SQL-based distribution algorithm using PostgreSQL array overlap (&&) to match articles with user portfolios and followed topics. Flutter news feed screen with sentiment color indicators, AI-generated summaries, asset impact tags, category filters, and pull-to-refresh.

## Alert System & Push Notifications
Impact alerts triggered by AI severity >= 7 during news processing. Volatility alerts triggered by >= 7% price change in 24 hours via Yahoo Finance monitoring. Firebase Cloud Messaging integration for push notifications. Single notification per article per user with highest-severity asset highlighted.

## Report Generation (Daily/Weekly)
Two-tier report system: Tier 1 shared AI asset reports generated by GPT-4o-mini, Tier 2 personalized portfolio calculations by Python. Celery scheduled tasks for daily (6PM) and weekly (Sunday 8PM) generation. In-app report viewing with on-demand generation option.

## Dashboard & UI Polish
Portfolio dashboard with total value, daily change, asset cards, allocation chart (fl_chart), alerts preview, and news preview. Material Design consistency, sentiment color coding, offline cache display with SQLite, and overall UI refinement.

## Testing & Bug Fixes
Unit tests (pytest for backend, Flutter Test for mobile), integration tests for API endpoints and external services, AI-specific tests (JSON validation, asset pool verification, severity range), and end-to-end flow verification.

## Performance Optimization & Security Review
API response time optimization, database query tuning, caching strategy for frequently accessed data, security audit (input validation, SQL injection prevention, API key management), and load testing with concurrent users.

## Deployment & Documentation
Backend deployment to Railway/Heroku with Gunicorn, APK build and distribution, environment variable configuration, final documentation updates, and thesis report preparation.
